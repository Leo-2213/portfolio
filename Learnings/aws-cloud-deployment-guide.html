🖥️ Compute Services

💻
EC2 (Elastic Compute Cloud)
Virtual Machines
Resizable virtual servers for running applications. Complete control over OS, middleware, and application stack.
✓ Best For
Traditional applications requiring full OS control, long-running services, high CPU/memory needs, legacy software
IaaS High Control Pay-as-you-go
📈
Auto Scaling Group (ASG)
Dynamic Scaling
Automatically scales EC2 instances up or down based on demand, CPU metrics, custom metrics, or schedule.
✓ Best For
Variable traffic patterns, handling traffic spikes, cost optimization, zero-downtime deployments
Scalability Cost Optimization
🔧
Elastic Beanstalk
Managed Platform
Platform-as-a-Service solution that handles deployment, scaling, and operations. Upload code and let AWS manage it.
✓ Best For
Web applications, microservices, quick deployments, teams wanting PaaS simplicity without Lambda constraints
PaaS Quick Deploy Multi-lang

🐳 Container & Orchestration

⚙️
ECS (Elastic Container Service)
Docker Container Orchestration
AWS-native container orchestration. Simplified container management on EC2 or Fargate with full AWS integration.
✓ Best For
Docker applications, AWS-first organizations, simpler than Kubernetes, tight AWS service integration
Containers AWS Native Orchestration
☁️
Fargate
Serverless Containers
Serverless compute for containers. Run Docker containers without managing underlying EC2 servers.
✓ Best For
Containerized apps wanting "serverless" benefits, avoiding infrastructure management, variable workloads
Serverless Containers No Ops
☸️
EKS (Elastic Kubernetes Service)
Kubernetes on AWS
Managed Kubernetes service. Full Kubernetes power with AWS management for control plane and integrations.
✓ Best For
Kubernetes expertise exists, multi-cloud strategy, complex microservices, standard Kubernetes tooling needed
Kubernetes Complex Apps Multi-cloud

⚡ Serverless Compute

λ
Lambda
Serverless Functions
Event-driven serverless functions. Pay only for execution time, automatic scaling, no infrastructure management.
✓ Best For
Event-driven workloads, microservices, APIs, data processing, scheduled tasks, bursty traffic patterns
Serverless Event-driven Cost-effective
🔌
API Gateway
API Management & Creation
Create, publish, maintain REST/WebSocket APIs. Integrates with Lambda, EC2, and other services with throttling and authentication.
✓ Best For
REST APIs, microservices interfaces, mobile backends, serverless API patterns, API versioning and management
API Integration Management

💾 Storage Services

📦
S3 (Simple Storage Service)
Object Storage
Unlimited object storage in the cloud. Perfect for backups, static websites, media files, logs, and data lakes.
✓ Best For
Static files, backups, archives, media distribution, big data processing, application logs, data lakes
Object Storage Scalable Durable
💿
EBS (Elastic Block Store)
Block Storage for EC2
High-performance block storage volumes for EC2 instances. Supports snapshots and multiple volume types for different performance needs.
✓ Best For
EC2 instance storage, databases on EC2, transactional workloads, high IOPS requirements, persistent storage
Block Storage EC2 Attached High Performance
🗂️
EFS (Elastic File System)
Shared Network File System
Managed NFS file system accessible from multiple EC2 instances simultaneously. Auto-scales with usage.
✓ Best For
Shared file access, big data processing, content management, machine learning training, multi-instance workloads
File Storage Shared Access Scalable

🗄️ Database Services

📊
RDS (Relational Database Service)
Managed SQL Databases
Managed relational databases. Supports MySQL, PostgreSQL, MariaDB, Oracle, SQL Server with automatic backups and failover.
✓ Best For
Traditional SQL applications, relational data, multi-row transactions, ACID compliance needed, legacy apps
SQL Managed HA/DR
✨
Aurora
High-Performance SQL
AWS-optimized relational database compatible with MySQL/PostgreSQL. Higher performance at lower cost with advanced features.
✓ Best For
High-traffic applications, mission-critical databases, needing superior performance, auto-scaling read replicas
High Performance Auto-scaling Distributed
⚡
DynamoDB
NoSQL Database
Fully managed NoSQL database. Single-digit millisecond latency, seamless scaling, on-demand or provisioned capacity.
✓ Best For
Real-time applications, mobile backends, IoT data, user sessions, leaderboards, flexible schema needs
NoSQL Low Latency Serverless
🔗
RDS Proxy
Database Connection Pooling
Managed database connection proxy. Reduces database load, improves resilience, enables connection pooling.
✓ Best For
Lambda/serverless heavy workloads, connection pool exhaustion issues, improving application resilience
Connection Pool Resilience Serverless

🌐 Networking & Load Balancing

⚖️
ALB (Application Load Balancer)
Layer 7 Load Balancing
Application-aware load balancer. Routes based on hostnames, paths, protocols. Perfect for microservices architectures.
✓ Best For
Microservices, containerized apps, path-based routing, hostname-based routing, WebSocket support needed
Load Balancer Layer 7 Microservices
⚡
NLB (Network Load Balancer)
Layer 4 Ultra-High Performance
Ultra-high performance network load balancer. Handles millions of requests per second with extreme low latency.
✓ Best For
Extreme throughput, gaming, real-time communications, extreme latency sensitivity, non-HTTP protocols
Load Balancer Layer 4 Ultra-Performance
🗺️
VPC, Regions & Availability Zones
Network & Global Infrastructure
VPC provides isolated network environments. Regions are geographic areas, Availability Zones are isolated data centers within regions.
✓ Best For
Multi-AZ deployment for HA, multi-region for DR, compliance in specific regions, latency optimization globally
Networking High Availability Disaster Recovery

🎯 Decision Guide by Use Case

📱 Building a Modern Web Application

Recommended Architecture: ALB → ECS/Fargate + RDS Aurora → CloudWatch

  • Frontend: S3 + CloudFront for static assets
  • Backend: ECS on Fargate for containerized API servers
  • Database: Aurora for relational data with auto-scaling read replicas
  • Load Balancing: ALB for path/hostname-based routing
  • Monitoring: CloudWatch for logs and metrics

⚡ Building Serverless Microservices

Recommended Architecture: API Gateway → Lambda → DynamoDB → CloudWatch

  • APIs: API Gateway exposes HTTP endpoints
  • Compute: Lambda functions for business logic
  • Data: DynamoDB for fast, scalable NoSQL storage
  • Async Processing: SQS/SNS for event-driven patterns
  • Cost: Pay only for actual execution, no idle infrastructure

📊 Running Enterprise Legacy Application

Recommended Architecture: EC2 + RDS + ASG + ALB

  • Compute: EC2 instances for application servers
  • Database: RDS for SQL database with managed backups
  • Scaling: Auto Scaling Group for horizontal scaling
  • Load Balancing: ALB to distribute traffic
  • Networking: Multi-AZ deployment for high availability

🐳 Kubernetes-First Organization

Recommended Architecture: EKS (Kubernetes) + ECR + RDS Aurora

  • Orchestration: EKS for managed Kubernetes cluster
  • Container Images: ECR to store Docker images
  • Multi-region: Deploy EKS clusters across regions
  • Database: Aurora with cross-region replicas
  • Networking: VPC with custom networking for pod communication

📈 High-Traffic Real-Time Application

Recommended Architecture: NLB → EC2 (ASG) + ElastiCache + DynamoDB

  • Load Balancing: NLB for ultra-low latency
  • Compute: EC2 instances with Auto Scaling
  • Caching: ElastiCache (Redis/Memcached) for sub-millisecond response times
  • Database: DynamoDB for single-digit millisecond latency
  • Monitoring: CloudWatch + X-Ray for performance tracking

🌍 Global Content Distribution

Recommended Architecture: S3 + CloudFront + Route 53

  • Storage: S3 for global object storage
  • CDN: CloudFront for global edge distribution
  • DNS: Route 53 for geographic routing
  • Origin Shield: Extra caching layer for protection
  • Cost: Extremely cost-effective for static content

🤖 Machine Learning & Data Processing

Recommended Architecture: Lambda/Batch → S3 → RDS/DynamoDB

  • Compute: Lambda for small jobs, Batch for large-scale processing
  • Storage: S3 for data lake and model storage
  • Database: RDS/DynamoDB for training data and results
  • Training: SageMaker for ML model training and deployment
  • Scalability: Automatic scaling without managing infrastructure
